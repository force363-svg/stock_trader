TECHNICAL DETAILS
================================================================================

Threading Model:
- Main thread: PyQt5 UI, user interactions
- Worker thread: Window manipulation, HTS input (keeps UI responsive)
- Proper closure capture prevents variable pollution between calls

Window Activation:
- Uses ctypes to find window by title containing "투혼"
- AttachThreadInput for proper thread context (Windows API best practice)
- ShowWindow(SW_RESTORE) -> BringWindowToTop -> SetForegroundWindow sequence
- Timing delays between operations for stability

Input Method:
- pyautogui.typewrite() for numeric stock codes
- Fallback: clipboard if pyautogui unavailable
- 0.03s interval between keystrokes for reliability

Error Handling:
- Graceful degradation if HTS not running
- Clear error messages in log area
- No silent failures (try-except with logging)
