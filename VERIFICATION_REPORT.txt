After (lines 1188-1228):
  def _activate_window(self, hwnd):
      if not hwnd:
          return
      
      try:
          user32 = ctypes.windll.user32
          kernel32 = ctypes.windll.kernel32
          
          current_thread = kernel32.GetCurrentThreadId()
          fg_hwnd = user32.GetForegroundWindow()
          fg_thread = ctypes.c_ulong()  # ✅ Proper ctypes type
          user32.GetWindowThreadProcessId(fg_hwnd, ctypes.byref(fg_thread))
          fg_thread_id = fg_thread.value  # ✅ Extract safely
          
          if current_thread != fg_thread_id:
              user32.AttachThreadInput(current_thread, fg_thread_id, True)
              time.sleep(0.05)  # ✅ Timing
          
          user32.ShowWindow(hwnd, 9)
          time.sleep(0.1)
          user32.BringWindowToTop(hwnd)
          time.sleep(0.1)
          user32.SetForegroundWindow(hwnd)
          time.sleep(0.1)
          
          if current_thread != fg_thread_id:
              user32.AttachThreadInput(current_thread, fg_thread_id, False)
      
      except Exception as e:
          print(f"Error activating window: {e}")  # ✅ Error handling

Improvements:
  1. Input validation: if not hwnd: return
  2. Proper ctypes types: ctypes.c_ulong() + ctypes.byref()
  3. Safe value extraction: fg_thread.value
  4. Timing between operations: time.sleep(0.1)
  5. Error handling: try-except wrapping
  6. Error logging: print(f"Error: {e}")

THREADING MODEL
================================================================================

Main Thread (UI Thread):
  1. User double-clicks stock in table
  2. PyQt5 calls cellDoubleClicked signal handler
  3. _on_holdings_click/search_click/related_click executes
  4. _send_to_hts() executes in main thread (brief)
  5. Creates and starts worker thread
  6. Returns immediately (UI stays responsive)
  7. Continues event processing

Worker Thread:
  1. Receives captured variables (code, name, timestamp)
  2. Calls _activate_window(hwnd)
  3. Sleeps 0.5 seconds
  4. Types stock code using pyautogui
  5. Presses Enter
  6. Returns (worker thread terminates)
  7. HTS receives input and processes

No communication between threads after start = no race conditions.
