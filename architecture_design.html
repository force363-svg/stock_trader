Thread Architecture

  • pyqtSignal 수신 -> 테이블/차트 업데이트
  • ai_signals.json 파일 감시 (mtime 체크, 3초)
  • 설정 다이얼로그, 사용자 입력 처리
^ pyqtSignal
^ pyqtSignal
^ file read
^ pyqtSignal
WORKER

xingAPI Worker

  • COM 전용 스레드 (CoInitialize)
  • 로그인 / 세션 관리
  • t1857 ACF 조건검색 (30초)
  • t0424 보유종목 (3초)
  • t1511 업종지수 (30분)
COM Thread

Market Worker

  • LS REST API 전용
  • 테마 / 업종 데이터 (5분)
  • 섹터 지수 갱신
  • 백그라운드 데이터 수집
REST Thread

AI Scanner

  • LS REST API (데이터 수집)
  • 종목 필터링 (등락률 2~9%)
  • 조건별 점수 산출 (scorer)
  • ai_signals.json 독점 기록
Scanner Thread

Trade Worker

  • LS REST API (주문 전용)
  • BUY 신호 수신 -> 매수 주문
  • SELL 신호 수신 -> 매도 주문
  • 미체결/체결 관리

Design Rules

UI = 표시 전용

Main Thread에서는 API 호출 절대 금지.
모든 데이터는 pyqtSignal로 수신.
파일 읽기(ai_signals.json)만 허용.

Worker = 데이터 소유

각 Worker는 자기 데이터만 기록.
xingAPI: holdings_cache, acf_list
AI Scanner: ai_signals.json (독점)

실전투자 수정 금지

main.py (실전투자) 절대 수정하지 않음.
새 파일(mock_main.py, mock_workers.py)로
모의투자 전용 프로그램 구성.

COM 스레드 격리

xingAPI COM 객체는 생성한 스레드에서만 사용.
CoInitialize/CoUninitialize 필수.
다른 스레드에서 COM 접근 금지.

AI Scoring Architecture

PRIMARY (Gate)

  • 60분봉 기준 3개 조건
  • RSI (40~70 범위)
  • MACD (비례가중치)
  • 볼린저밴드 (위치 판단)

모든 PRIMARY >= 60 이어야 통과

→

BONUS (Additive)

  • 단기눌림목 패턴
  • 스퀴즈 브레이크아웃
  • 박스권 돌파
  • 눌림목 + 이격도

max(0, (avg-50)*0.3) 가산

→

FINAL SCORE

  • >= 75: BUY 신호
  • >= 55: HOLD 신호
  • < 55: WATCH 신호

PRIMARY base + BONUS add
